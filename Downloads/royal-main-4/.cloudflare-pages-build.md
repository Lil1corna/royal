# ⚡ Cloudflare Pages — Настройка в Dashboard

## 🎯 Шаг 1: Build Settings

Откройте ваш проект в Cloudflare Pages Dashboard → Settings → Builds & deployments

### Используйте эти настройки:

```
Framework preset: Next.js
Build command: npx @cloudflare/next-on-pages
Build output directory: .vercel/output/static
Root directory: (оставьте пустым)
```

**ВАЖНО:** Если у вас уже был другой build command, обязательно измените его на `npx @cloudflare/next-on-pages`

---

## 🔐 Шаг 2: Environment Variables

Settings → Environment variables → Production

Добавьте эти переменные:

```bash
NODE_VERSION=20

# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGci...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGci...

# Site URL (замените на ваш домен после деплоя)
NEXT_PUBLIC_SITE_URL=https://royalmatrasaz.pages.dev

# Google OAuth (если используете)
GOOGLE_CLIENT_ID=xxxxx.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=GOCSPX-xxxxx
```

---

## 🚀 Шаг 3: Retry Deployment

После изменения настроек:

1. Идите в **Deployments**
2. Найдите последний failed deployment
3. Нажмите **Retry deployment**

Или запустите новый деплой:

```bash
git commit --allow-empty -m "chore: trigger Cloudflare rebuild"
git push origin main
```

---

## ✅ Что должно произойти

Build займёт ~2-3 минуты:

```
✓ Collecting build output
✓ Uploading build output
✓ Deploying to Cloudflare's global network
✓ Success! Deployed to https://royalmatrasaz.pages.dev
```

---

## ⚠️ Если всё равно ошибки

### Вариант 1: Очистить кэш

Settings → Builds & deployments → Clear build cache → Retry deployment

### Вариант 2: Пересоздать проект

Если ничего не помогает:

1. Удалите проект в Cloudflare Pages
2. Создайте новый проект
3. Подключите тот же GitHub репозиторий
4. Используйте настройки выше с самого начала

---

## 📊 Проверка после деплоя

1. Откройте ваш сайт: `https://royalmatrasaz.pages.dev`
2. Проверьте, что страницы загружаются
3. Проверьте авторизацию (если настроена)
4. Проверьте функционал корзины/wishlist
