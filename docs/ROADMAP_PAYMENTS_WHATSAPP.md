# Дополнения к сайту, оплата, WhatsApp

## Что ещё можно добавить

- Онлайн-оплата (Stripe / локальный эквайринг), счёт-фактура PDF
- SMS при смене статуса заказа
- Push-уведомления (PWA)
- Отзывы и рейтинг товаров
- Поиск и фильтры в каталоге (уже частично есть)
- Мультивалюта, налоги
- Интеграция с бухгалтерией / 1С

## Способы оплаты (методы)

| Метод | Для кого | Примечание |
|--------|-----------|------------|
| **Stripe** | Международные карты | Популярно, нужен аккаунт Stripe, вебхуки |
| **PayPal** | Международные покупатели | Официальный SDK |
| **Локальный банк / эквайринг** | Азербайджан | Зависит от банка (API, редирект на оплату) |
| **Наложенный платёж / при получении** | Сейчас по сути так | Без интеграции |
| **Apple Pay / Google Pay** | Часто через Stripe | Доп. настройка |

Реализация: создать сессию оплаты на сервере (Route Handler / Edge Function), после успеха — вебхук подтверждает заказ и статус `paid`.

## WhatsApp: уведомление курьеру при «в доставке»

**Вариант A — вебхук (уже в коде)**  
При переводе заказа в статус **Yolda / in_delivery** приложение отправляет `POST` на URL из переменной `DELIVERY_WEBHOOK_URL` с JSON:

```json
{
  "event": "order_in_delivery",
  "order_id": "uuid",
  "address": "текст адреса",
  "notes": "Tel: +994... | ... | Koordinat: 40.xxx,49.xxx",
  "total_price": 123,
  "maps_link": "https://www.google.com/maps?q=40.xxx,49.xxx",
  "customer_phone": "+994..."
}
```

Дальше в **n8n**, **Make**, **Zapier** или своём сервере можно вызвать **WhatsApp Cloud API** (Meta) или **Twilio WhatsApp** и отправить сообщение курьеру.

**Вариант B — WhatsApp Business API (официально)**  
1. Meta for Developers → приложение → WhatsApp.  
2. Получить токен и ID телефона отправителя.  
3. Шаблонные сообщения для первого контакта; дальше — сессионные сообщения.  
4. Номер курьера — получатель в формате `994XXXXXXXXX`.

**Вариант C — без бота, только ссылка**  
Кнопка в админке «Открыть WhatsApp» с `https://wa.me/994XXXXXXXXX?text=...` (текст с адресом и ссылкой на карту) — пользователь сам нажимает, сообщение не уходит автоматически.

### Переменные окружения (Vercel / .env.local)

```
DELIVERY_WEBHOOK_URL=https://your-n8n-or-server.com/webhook/delivery
DELIVERY_WEBHOOK_SECRET=optional-shared-secret
```

Проверка на приёме: заголовок `X-Webhook-Secret` должен совпадать с `DELIVERY_WEBHOOK_SECRET`.
